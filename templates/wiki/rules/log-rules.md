# Log Rules

本檔只定義 `wiki/log.md` 的更新規則，不處理任務分類、資料讀取主流程或最終對外回覆格式。

## 何時必須讀取

遇到以下情況時，必須讀取並遵循本檔：

1. 任務明確要求更新 `wiki/log.md`。
2. 任務屬於 Worker、webhook、agent、排程或其他系統自動執行流程。
3. 其他 task rule 指示本次任務完成後需要留下 log 紀錄。

## 核心原則

1. `wiki/log.md` 是時間序列紀錄。
2. 新 log 一律附加在 `wiki/log.md` 檔案尾端，不可插入中間，不可改寫舊紀錄。
3. 若 task rule 要求寫 log，更新 `wiki/log.md` 就是流程的一部分，不是可選動作。
4. 若工具或權限不足而無法寫入 log，必須在最終回覆中明確說明 `log 尚未寫入` 與原因，不可假裝已完成。
5. 若本次只是純人工對話，且未經 Worker、webhook、agent 或排程流程執行，則不應為了查詢回覆額外改寫 `wiki/log.md`。

## 寫入方式

1. 追加 log 時，優先使用適合尾端附加的寫檔工具。
2. 若 task rule 有指定 log 格式，應遵循該格式。
3. 若同一次任務同時更新多個 wiki 檔案與 `wiki/log.md`，必須先完成主要寫入，再追加 log。
4. log 內容必須只根據本次實際執行的讀檔、寫檔與輸出結果，不可把未實際執行的動作寫進 log。
